File Path System

Step 1: The First Principle

A website is not one single file. It's a collection of different files (HTML, CSS, JavaScript, images, videos, fonts) that work together. These files are organized into folders.

Step 2: The Core Problem

If your index.html needs to display an image named logo.png, how does the HTML file tell the browser where to find it?

The browser needs an exact address to locate the file. A file path is that address.

A) Relative File Paths

Relative paths start from the location of the file you are currently in. You'll use these most of the time.

my-website/
├── index.html
├── about.html
├── images/
│   ├── logo.png
│   └── hero.jpg
└── pages/
    ├── contact.html
    └── terms.html

B) Absolute File Paths

Absolute paths give the full URL to a resource. Use them only for external resources.

Search on Google

Why Not Use Absolute Paths for Local Files?

Copying paths like C:/Users/Arjun/Desktop/my-website/images/logo.png will only work on your computer. On a web server, it breaks.

Summary / Rules of Thumb

  1. For your own files, ALWAYS use Relative Paths.
  2. For other websites' files, ALWAYS use Absolute Paths.

Key Tags Explained

  • <!DOCTYPE html> - Defines HTML5 standard.
  • <html lang="en"> - Sets page language for accessibility and SEO.
  • <head> - Metadata for the browser, not visible content.
  • <meta charset="UTF-8"> - Ensures correct text encoding.
  • <meta name="viewport"> - Enables responsive design for mobile devices.
  • <title> - Name shown in browser tabs and bookmarks.
  • <body> - Contains all visible content.
